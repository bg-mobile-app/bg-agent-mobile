I have identified the issue! It is caused by the backend's localization logic combined with how Dart (Flutter's language) parses numbers.

When you request the data in Bangla, the backend translates the numbers in the JSON into Bangla numerals (e.g., instead of "18", it sends "১৮").

The web application doesn't have an issue with this because it treats them as plain strings and just displays them on the screen. However, our Flutter app takes these values and specifically parses them into integers (numbers) to be properly handled by our models.

Dart's int.tryParse only understands standard English/ASCII digits (0-9). When it tries to parse "১৮", it fails, resulting in null, which the app safely falls back to 0 (which is why they were hidden/not rendering).

The Fix
I have just updated the model in 
work_permit_details.dart
 to include a conversion utility. Now, whenever the app receives a string to parse into a number, it will first translate any Bangla numerals (০-৯) back into English digits (0-9) before attempting to parse them.

This fixes the bug not just for minAge and maxAge, but for any numeric fields coming from the API that might have been automatically translated into Bangla digits!
